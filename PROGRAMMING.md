# Programmation pour les 6-10 ans

Ce guide explique la programmation avec des mots simples, des exemples concrets et des mini-défis amusants.

## 1) C’est quoi la programmation ?

Programmer, c’est **donner des instructions à un ordinateur**.

Un ordinateur est très fort pour calculer vite, mais il a besoin qu’on lui dise **exactement** quoi faire.

### Exemple de la vie de tous les jours

Imagine que tu expliques à un robot comment se brosser les dents :

1. Prendre la brosse à dents.
2. Mettre du dentifrice.
3. Brosser en haut.
4. Brosser en bas.
5. Rincer.

C’est déjà un **petit programme** !

## 2) Les 3 idées magiques

### A. Les instructions

Une instruction = une action.

Exemples :

- avancer
- tourner
- dire « Bonjour »

### B. Les boucles

Une boucle, c’est **répéter**.

Exemple :

- sauter 5 fois

Au lieu d’écrire “sauter” 5 fois, on dit : “répète 5 fois”.

### C. Les conditions

Une condition, c’est “**si… alors…**”.

Exemple :

- Si il pleut, alors je prends un parapluie.

En code :

- Si le personnage touche un mur, alors il rebondit.

## 3) Le code, c’est comme des LEGO

Quand tu construis avec des LEGO, tu assembles des briques.

En programmation, tu assembles des blocs d’instructions :

- un bloc pour avancer,
- un bloc pour parler,
- un bloc pour répéter,
- un bloc pour choisir (si/alors).

Petit à petit, tu construis un jeu, une animation ou une histoire.

## 4) Un mini-exemple (très simple)

Objectif : faire dire bonjour à un personnage.

1. Quand on clique sur le drapeau vert,
2. le personnage dit “Bonjour !”,
3. puis il avance de 10 pas.

Bravo : tu viens de créer ton premier programme.

## 5) Les erreurs sont normales

En programmation, les erreurs s’appellent parfois des **bugs**.

Ce n’est pas grave :

1. On teste.
2. On observe ce qui ne marche pas.
3. On corrige.
4. On reteste.

Les programmeurs font ça tous les jours.

## 6) Mini-défis à faire

### Défi 1 — Robot danseur

Fais un personnage qui :

- avance,
- tourne,
- répète ce mouvement 4 fois.

### Défi 2 — Jeu de devinette

Fais poser une question au joueur.

- Si la réponse est correcte : “Bravo !”
- Sinon : “Essaie encore !”

### Défi 3 — Histoire animée

Crée 2 personnages qui parlent à tour de rôle.

## 7) Outils conseillés

Pour commencer en douceur :

- **Scratch** (avec des blocs visuels)
- **Blockly** (même idée, très visuel)

Pas besoin d’écrire du code compliqué au début.

## 8) Message important

Tu n’as pas besoin d’être “un génie” pour programmer.

Il faut surtout :

- être curieux,
- essayer,
- se tromper,
- recommencer.

Et surtout : **s’amuser** 🎉
